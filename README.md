# Bank Customer Churn Analysis

## Task 4 – Data Storytelling & Statistical Validation

## 📌 Project Overview

This project focuses on analyzing customer churn behavior in the banking sector using Data Storytelling techniques and Statistical Validation methods. The objective is to transform raw banking data into meaningful business insights through visual analytics and hypothesis testing.

The project was implemented using Python in Google Colab with libraries such as Pandas, Matplotlib, Seaborn, and SciPy.

---

## 🎯 Objectives

* Analyze customer churn patterns
* Create meaningful visual stories from data
* Perform statistical hypothesis testing
* Validate business assumptions using statistical methods
* Generate actionable business insights

---

## 📊 Dataset Information

* Dataset Name: Cleaned_Bank_Churn.csv
* Domain: Banking
* Records: Customer banking details
* Features Included:

  * Credit Score
  * Gender
  * Age
  * Balance
  * Estimated Salary
  * Tenure
  * Number of Products
  * Exited (Churn Status)

---

## 📊 Statistical Validation Performed

### 1. T-Test

Used to compare the age difference between churned and non-churned customers.

### 2. Chi-Square Test

Used to determine the relationship between gender and customer churn.

### 3. Correlation Analysis

Used to identify relationships between balance and estimated salary.

---

## 🔍 Key Insights

* Older customers showed a higher probability of churn.
* Customers with larger account balances were more likely to leave the bank.
* Gender demonstrated some association with churn behavior.
* Balance and estimated salary had weak correlation.
* Statistical testing validated several business assumptions.

---

## 💡 Business Recommendations

* Implement personalized retention strategies for high-risk customers.
* Improve engagement for older customers with tailored financial products.
* Monitor high-balance customers closely to reduce churn probability.
* Use predictive analytics for proactive customer retention.
